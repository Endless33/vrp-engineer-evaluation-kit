# Frequently Asked Questions

# What is this repository?

The VRP Engineer Evaluation Kit is a public engineering evaluation environment for the Veil Routing Protocol (VRP).

It is intended to help engineers independently evaluate publicly documented behavior through reproducible testing.

---

# Is this the complete VRP implementation?

No.

This repository intentionally excludes:

- protected runtime implementation;
- proprietary algorithms;
- confidential engineering logic;
- private authority mechanisms;
- production infrastructure;
- private cryptographic material.

---

# Can this repository be used in production?

No.

It is designed exclusively for engineering evaluation, experimentation, and validation.

---

# What can be evaluated?

Depending on the published release, engineers may evaluate:

- documented protocol behavior;
- engineering workflows;
- validation procedures;
- exported evidence;
- reproducibility of published scenarios.

---

# Why is the protected runtime not included?

The purpose of this repository is independent engineering validation.

The protected runtime remains outside the public evaluation environment.

This separation allows engineers to validate documented behavior without exposing confidential implementation details.

---

# Can I submit bug reports?

Yes.

Reproducible engineering reports are welcome.

Please include:

- operating system;
- software versions;
- environment description;
- reproduction steps;
- generated logs;
- exported evidence;
- engineering observations.

---

# Can I contribute?

Yes.

Documentation improvements, reproducible evaluation scenarios, automation improvements, and engineering feedback are encouraged.

Please read CONTRIBUTING.md before submitting changes.

---

# Where should engineering discussions take place?

Technical discussions should focus on:

- reproducibility;
- engineering evidence;
- validation methodology;
- measurable observations;
- documentation improvements.

# Final Note

Engineering confidence is built through independent verification.

The purpose of this repository is to make that verification as transparent, reproducible, and technically rigorous as possible.
