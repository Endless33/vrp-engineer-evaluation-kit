# Contributing

Thank you for your interest in the VRP Engineer Evaluation Kit.

This repository exists to support independent engineering evaluation through reproducible technical evidence.

## Goals

Contributions should improve one or more of the following:

- evaluation quality;
- reproducibility;
- documentation;
- test scenarios;
- reporting;
- engineering clarity.

## Good Contributions

Examples include:

- improving documentation;
- fixing documentation errors;
- adding reproducible evaluation scenarios;
- improving automation;
- improving report generation;
- improving engineering usability.

## Pull Requests

Please ensure that every pull request:

- has a clear engineering purpose;
- remains reproducible;
- does not introduce proprietary VRP implementation;
- does not expose confidential runtime logic;
- includes documentation where appropriate.

## Issues

When reporting a problem, include:

- operating system;
- environment details;
- reproduction steps;
- expected behavior;
- observed behavior;
- logs;
- generated reports, if available.

Issues without sufficient technical information may not be reproducible.

## Scope

This repository intentionally excludes:

- protected runtime source code;
- proprietary algorithms;
- confidential implementation details;
- private cryptographic material;
- production deployment logic.

## Engineering First

Engineering discussions should focus on technical evidence.

Questions, constructive criticism, improvements, and reproducible validation are always welcome.

Thank you for helping improve independent engineering validation.
